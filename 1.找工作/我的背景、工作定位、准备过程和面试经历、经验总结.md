# 我的背景、工作定位、准备过程和面试经历、经验总结

一、背景

​	双非本硕、考了两次985硕士都以差2分进复试告终，最后调剂到另一所综合实力稍强，但计算机一般双非读硕士，真的很无奈，万般难受。本硕都是软件工程专业，24届应届毕业生。虽然两次全力以赴的考研都以失败告终，感觉差了一点运气和心态的调整，成了我非常难忘的经历，但我并没有摆烂的想法，一直认为只要自己足够努力，也是可以的。

二、工作定位

​	本科阶段完全没有实习的意识（身处的环境氛围还是有差距，认知不够），等到硕士时候，自己就会点python和自学的算法相关知识。听说开发实习不好找，也不会，就直接投了测试实习，心想只要有一份实习我就去，2023年春天找了一份芯片小厂的测试实习（期间也面了滴滴、网易什么的，机会还是有的，就是看自己能不能把握住，现在回想起来，其实找实习难度比秋招简单很多，然而，我当时真的是什么都不会啊，八股文都知道的太少了，也不知道自己这些年学了啥），实习干了3个月就回来了，回学校是因为自己没有论文且实习内容比较简单，感觉学不到东西了，就回来了，但是待遇还是不错的，现在想想如果一直实习能留在那，也很不错，毕竟工资高，不加班，当时还是一门心思想学东西，有点天真（当然也很感谢这段实习经历，是我人生中的第一个次工作，学到了很多知识，包括Linux服务器，在公司学东西真的非常快，比在学校快多了，而且还是志趣相投的同学，真的不一样），但如果让我再选择一回，我还是会做同样的选择，选择离开，因为学不到脚本的编写，只是会点linux命令，后来就成了机械人，每次版本测试都是重复操作，没有涉及到自动化脚本、自动化框架的学习，也成为了我后来找工作时介绍实习时的硬伤，无论怎么包装我的实习，当面试官一问你怎么测试你的功能时，我就只能介绍功能，不能介绍具体的自动化处理逻辑，所以实习短短不到3个月离开不后悔！

​	回学校也没搞出什么东西，论文还是搞不出，实验太难弄了，完全自己一个人弄，什么都自己，（反正我们组就这样），但我也自己学了很多东西，提高了python编程能力，深度学习项目的运行环境、复现、代码的拼接我都能完成，这是是我天天在工位一点点扣出来的。

​	以为上了研究生就能搞算法了，选的方向也是自然语言处理，但是了解到，双非硕士不建议搞算法，自己也确实没资源、没背景、对自己没有信心，就这样放弃了（但是，现在我觉得如果你真想搞算法，就一直搞下去，不要听这个的听那个的，只要你自己付出足够的时间，采用正确的方法，一定可以的！）。后来，就寻思干开发、测开啥的，因为自己对C语言有基础且会一点python，所以定位就是C++开发工程师、测试开发工程师，又因为自己又一份测试的实习经历，寻思先找测开的工作吧，如果能找到测开，我再找找C++开发的工作。但是现在想想，测开好像也没比开发少什么，还不如直接冲开发了，然后可以向下兼容，向下兼容容易，向上就难了，而且准备的时候，按照开发的准备会学到很多东西，但是按照测试的准备的话就容易没有动力（我刚开始找测开的简历是没有开发项目的，只有一个实习经历项目和一个研究经历，但是后来发现，只有实习经历远远不够，还得有开发的项目经历，后来就补了一个C++ webserver服务器项目，再后来春招的时候我又补了一个卡哥的简单版本的STL项目并自己加了迭代器和算法，感觉简历才充实一点，现在想想如果当时就按照开发的标准去准备，可能更好一点儿）。

三、准备过程

​	实习回来都23年4月份了，才开始准备，之前刷过1/4的算法题，因此我直接看八股了（八股看不懂的地方，我再去找资料，找论坛，博客啥的。）

准备分为几个部分：

1. 计算机基础

​	我是先看了卡哥的**《计算机基础学习路线》**，**《c++学习路线》**，还有**星球的录友的学习路线**后内容，了解学习资源在哪里，准备过程，并结合自己的目前的水平，还时不时的去回看学习路线和经验，哈哈。知道自己时间紧张，总怕学不完，我是直接看了八股的。当时也是很纠结，很焦虑，这么多，到底从哪里下手，其实不管哪个，只要你开始着手看了，心态也就平静下来了，看过一遍，慢慢的有底了，你的心态就不一样了，然后慢慢的所有的知识就都看过了，但到现在我也没有都看过一遍，哈哈，如果有时间还是系统学习。

​	不管时间紧不紧张，我觉得要有每天都在学习，都在进步的心态，时间放长远，既然选择了这个职业，以后时间长着呢，只要你有技术，有能力，你总会找到工作的。

- 数据结构：这个我没怎么复习，因为考研的时候学过两年，而且也不怎么问，**一般都是结合算法来学习**吧。

- 操作系统：卡哥推荐的书，太多了，有点害怕，哈哈（但是其实有时间还是要好好学习哪些书，我这种属于应付，对操作系统还是没有整体的认识），所以就小林coding、代码随想录八股文、阿秀的学习笔记面经、结合着看，在理解的基础上背下来。

- 数据库：

  1. 因为数据库真的几乎全忘了，所以还是认真的看了一遍《MySQL 必知必会》并做了笔记，方便以后复习，感觉好多了，有感觉了。然后就是看常考的面经，边理解，边查资料，边背的这种。

  2.  还有就是刷mysql的编程题，面试官会考一到mysql的题和算法的题，所以刷题很重要。

- 计算机网络：看了卡哥推荐的入门书《网络是怎么连接的》并做笔记，方便以后复习，从整体上有了一个了解，然后就看八股文，搜集面经，开始总结成笔记，背，计算机网络我觉得还是很好理解的。

2. 编程语言

​	因为我硕士期间是自然语言处理，会用到python，用python去复现代码，修改项目等，还是积累了一些编程能力的，而且测试开发一般也会用python，也看了廖雪峰的学习网站中的python基础，并去回看。

​	后来，我发现python其实更多的是类似脚本的作用吧，我觉得还不够，应该掌握一门更佳正规的语言。因为我考研时候自学过c++基础，也用c语言在考研期间写了很多的算法题去准备考试，对c++还是比较喜欢的，所以后来我就又开始学c++基础和相关八股文。我直接看的c++八股文，不懂的地方去c++ primer中去搜，网站中搜，理解清楚，反正就是哪里不会补那里了。

​	直接背c++八股也得有一定的基础知识，要不然都看不懂，但是背八股也让我面试时吃了很多亏，就是背八股不成体系，无论什么问题你只能回答一点点，不知道跟哪些知识连在一起，并且很慌，对基础知识掌握不牢固，你背八股很容易被面试官看出来，所以，**我觉得学习C++不能走捷径，一定要踏踏实实，从基础到高级特性，一定边写代码边实践边理解边做笔记，这样会越走越快，我这种方式只能是赌**。

4. 项目

  项目我准备的比较急，但是还是**希望能够一步一步的理解每一个细节，每一个细节都有可能会被问到，** 当然，我没有做到。

​	**牛客的C++ webserver**，现在还收费了，我看的时候都不收费... ... 已经烂大街了，但是自己确实啥都不会，所以只能先去看这个，边看视频边照着写代码，全下来就一周多的时间。之后我把课程资料都下载下来，后面不断的完善这个项目，对项目进行了重构，加入了定时器...**卡哥的知识星期里都有c++ web服务器的面经**。

​	这个项目虽然烂大街了，但是对于什么都不会的我来说，还是**让我对数据库、计算机网络、操作系统、Linux系统下的基于C++语言的基于socket网络编程等知识进行了一个串联**，收益很多。但是我觉得**只有这个项目，确实远远不够**，还得做一个你针对找工作的具体方向做的项目... ...。

​	所以后来我就又做了卡哥的**卡码网的《一个简单的手写版本的STL》**，并且自己**加了迭代器和算法**。这才让我的简历充实了一些，就这样我一边找工作，一遍给简历增加内容，不断的增加自己的知识，最重要的我觉得就是我没放弃，我边面试，边学习。（**卡码网还有其他的项目，我时间紧迫，选了比较简单的... ...**）

5. 实习

​我是误打误撞找到了一个测试的实习就去了，但是还好，算是个有名的中厂。这段实习经历对我之后找工作起了巨大的作用，如果没有这段实习经历，我可能到现在都找不到工作，**实习真的很重要**。
  
但是呢，这份实习**比测试强一点**，因为他需要Linux服务器上操作，**比测开差很多**，因为接触不到自动化脚本，然而，接触不到脚本、框架，在我秋招、春招的时候就没办法给面试官讲清楚某个功能是怎么通过编写脚本，处理的，而这正是**面试官最想知道的！... ... 就是你你实习时用了哪些语言、框架、处理逻辑是什么来处理你负责的功能模块的，而我却在实习时没接触到，真的很伤！** ... ... 其他都好包装，但是这个真的就没法包装。
  
所以**推荐，找实习一定要找一个测开的实习，能接触到自动化框架、脚本的那种**，当然如果实在找不到，可以去一个比较大，有名的厂；还可以**分两步，先找一个小厂的测试，再根据小厂的实习经历去找中大厂的实习**，这样会好找一些，然后最好时可以转正的那种，**通过找实习可以转正的这种方式，是我觉得最稳妥，最省心的找工作的方式，秋招的时候，大家都在找，有时候即使你简历很优秀，可能也过不去！所有方式方法很重要！还有一点就是找工作的时机也很重要，不要赶着人最多的时候找，除非你是985！**

​实习项目在面试中的准备有一些常考的问题（每一个问题都要事先总结好，并不断积累问题）：

主要考察的是你的 1. 测试思维（通过设计测试用例，来看你是否有发散思维、批判性思维等等...） 2. 表达能力（因为测试要和开发去battle），逻辑思维 3. 测试基础知识 4. 编程能力

1. 你能详细介绍一下你的实习经历吗，在其中你的任务有哪些，你负责哪个模块

（实习经历，你都负责什么，具体哪个模块，业务流程是什么?（字节））

答：根据star法则（情景-任务-行动-收获）的结构来回答，体现你的逻辑性。

2. 你说的是你们组的大概的处理流程，你能具体说说你负责哪个模块，具体什么功能，你的处理逻辑是什么吗？（这个不是简单说功能，而是你通过什么工具、什么自动化脚本，如果多你负责的测试的功能/模块进行处理的，测试的，面试官想了解你用了什么框架、什么语言、你的处理逻辑有没有问题等等）

8. 面试经历

  面试会从**自我介绍、八股基础知识、实习、项目、职业发展、意向城市及原因、性格特点**等方面进行提问，最后会**手撕代码**并**讲解过程**，可能还会**针对代码设计测试用例**，**手写sql语句**（sql有时候会考察你的**反向思维**，比如找出所有课程成绩大于80的同学名称，你可以直接通过只要有课程小于等于80就排出的方法来过滤），最后就是**反问环节，问一些技术栈，自己欠缺在哪里等问题**，每一个环节都应该准备一下。

  我有一些心得，想分享给大家，我觉得真的是，别人说1000遍，你都不明白，自己经历一遍，就立马明白了，该走的路，一步都不能少：

  1. 一定不要紧张，告诉自己**稳住，不要慌**，否则，准备了好多，但是发挥不出来，这真的太难受了，我就是每次都紧张，后来面多了，还好一些。面之前，告诉自己，稳住，不要慌。

  2. 我认为，自我介绍时一定要自然，不要死记硬背，要与面试官有眼神的交流，**真实、自然、有条理的介绍自己的工作内容即可。**

  3. 接着会针**对你的自我介绍中的内容**（有可能打断，直接问面试官感兴趣的部分），**和简历中的内容进行提问**。所以写在简历上的东西一定要会，一定要深刻了解，否则，指不定哪次面试，如果你准备不充分，就会尬住，当然，大家简历好像都有水分，这个自己把握。

  4.  面试官其实不想为难你，他只是想基于你的简历进行提问，看你是否如实的对你简历中的内容都掌握的，并且能够**有逻辑的、清晰的表达**出来，一方面体现了你的专业能力、一方面体现了一的表达能力。所以**一个很可能提到的问题，就是你投递的岗位的职责工作内容/要求 与 你简历中的你做的项目内容/工作 有交叉的部分，这个面试前一定要准备好，能回答上来，说明你对项目理解的不错，做了一些工作的**。
  
  5. 还有一种面试方式就是，面试官一般会由浅入深的对你进行提问，如果浅的都回答不上来，可能就会各种理由终止面试。 其实面试官并不想考你很难的东西，他只是**想通过某种方式了解你的知识深度，是否对核心概念真正掌握**，而**不是东背一下八股，西背一下八股**。[如果你是一个C++面试官，你会问哪些问题？ - 果冻虾仁的回答 - 知乎](https://www.zhihu.com/question/451327108/answer/2275638668)这个就是一个例子，我受益非浅，就是做什么事都不要好高骛远，还是要脚踏实地， ** 一定重视基础，一定重视手撕代码，如果连基础的代码都写不出来，肯定凉了**
